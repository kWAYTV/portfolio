Todo desarrollador tiene una carpeta. Ya sabes cuál.

`~/projects`, `~/code`, `~/dev`. Tal vez incluso `~/definitivamente-termino-este`.

Es donde los proyectos personales van a morir.

## Los residentes

Déjame presentarte algunos de los míos:

### 1. Los proyectos "construiré mi propio X"

- Una app de notas personalizada (porque Notion no era suficiente)
- Un rastreador de hábitos (lo usé 3 días)
- Un dashboard de finanzas personales (irónico, dado que gasté dinero en dominios para ello)

Estado: Último commit hace 8 meses. El README dice "próximamente."

### 2. El cementerio de tutoriales

Cursos a medias. Series de YouTube abandonadas. Esa compra de Udemy de 2019.

Tengo el 47% de un CLI en Rust en algún lugar. Imprime "Hola" pero no "Mundo." Casi.

### 3. Las ideas "esto me hará rico"

Un SaaS que resuelve un problema que nadie tiene. Completo con:
- Landing page ✓
- Formulario de lista de espera ✓
- Producto real ✗

La lista de espera tiene 3 inscripciones. Todas yo. Probando.

### 4. La app de tareas sobre-diseñada

El rito de paso de todo desarrollador. La mía tiene:
- API GraphQL
- Config de Kubernetes
- Pipeline CI/CD
- Sincronización en tiempo real por WebSocket

No puede marcar una tarea como completada. Esa función está "en el backlog."

## ¿Por qué mueren?

Las mismas razones siempre:

1. **Algo nuevo y brillante** - Empecé a aprender Go, abandoné el proyecto en Python
2. **Scope creep** - "¿Y si también planificara comidas?"
3. **El muro del 80%** - Funciones core listas, pero auth y deployment son aburridos
4. **La vida** - El trabajo se puso ocupado, toqué pasto, olvidé qué hacía el código

## Las mentiras que nos contamos

- "Volveré a esto"
- "El código está autodocumentado"
- "Recuerdo exactamente dónde lo dejé"
- "Esto es una experiencia de aprendizaje" (lo fue, pero aún así)

## ¿Deberías sentirte mal?

No.

Cada proyecto abandonado te enseñó algo. Ese CLI en Rust me enseñó lifetimes. El SaaS me enseñó que odio escribir copy para landing pages. La app de tareas me enseñó que Kubernetes es excesivo para todo.

El cementerio no es fracaso. Es un museo de curiosidad.

¿Este blog? También es un proyecto personal.

Vuelve en 6 meses a ver si se une a los demás.
